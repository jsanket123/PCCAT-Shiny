Frequently Asked Questions

Q1. How should I input my data?

Your input data file must be a CSV file that follows the simple yet strict rules given below:

  1. The CSV files must be in the form of Column A = SampleID (a unique value for every row), Column B = Group (logical grouping in your data), and Column C onwards = all the numerical variables. (If you are planning to include more categorical features then add them after the Group variable and before all the numerical featuresand indicate your numerical feature start column appropriately.)
  2. Your first row, Row 1 must have the Header Row as below. Actual data must start on Row 2.
  3. All laboratory analytical data should be in the exact same units (organics, inorganics, should be in consistent units, ppt, ppb, ppm).
  4. All laboratory analytical data non-detects must be treated in the exact same way, i.e., using zero (0) for ND, or 1/2 the detection limit, but you must be consistent throughout the data. Note: Do not use parameters that have more than 75% non-detects in your PCCAT CSV, as there is not enough differentiation if the parameter is mostly non-detect.
  5. All data fields must have a value. Do not leave a data field empty. See the example below:

    • Column A holds unique individual Sample Identifications. (Note: Descriptions are useful for special locations, such as “Horse Creek” in the example)
    • Column B holds Group Information, i.e., which Samples are Groundwater or Surface Water. Columns C through I are individual parameters in ng/l, with non-detects coded as zero (0).

For instance, Group can be location based (Groundwater/Surface Water, Field Blank, Upstream/Downstream) or year based (2002, 2007, 2020), or both (GW_2002, GW_2007, GW_2020, SW_2002, SW_2007, SW_2020). Try to use 2-6 groups to help in the visualization. If you do not include a Group in Column B of your CSV, you must indicate that your numerical features start in column 2 in the Data Input tab.

Q2. What are my data transformation options?

  • Use standardize if your data is normally distributed (typical for environmental data).
  • Use log-transform if you know your data is log normally distributed. Note: your variable values have to be positive to apply this transformation

Q3. What is principle component analysis?

  • Principle Component Analysis (PCA) is mostly used as a tool in exploratory data analysis and for making predictive models.
  • It is often used to visualize genetic distance and relatedness between populations (wiki).
  • A general overview of PCA can be found here (StatQuest 6 minute video): https://www.youtube.com/watch?v=HMOI_lkzW08

Q4. What is clustering analysis?

  • Cluster analysis or clustering is the task of grouping a set of objects in such a way that objects in the same group (called a cluster) are more similar (in some sense) to each other than to those in the other groups (clusters).
  • It is a main task of exploratory data mining, and a common technique for statistical data analysis, used in many fields, including pattern recognition, image analysis, information retrieval, bioinformatics, data compression, computer graphics, and machine learning (wiki).
  • A general overview can be found here (StatQuest 11 minute video): https://www.youtube.com/watch?v=7xHsRkOdVwo
